# Security Policy

## Supported Versions

Only the latest public release of VLC RPC is supported for security fixes.

## Reporting a Vulnerability

Please do not open a public issue for a security vulnerability.

If you find a vulnerability, contact the maintainer privately where possible, or use GitHub private vulnerability reporting if it is enabled for this repository.

Please include:

- A clear description of the issue
- Steps to reproduce
- Potential impact
- Any suggested fix

## Scope

Security-sensitive areas include:

- Discord IPC and OAuth handling
- AniList OAuth/token handling
- Local VLC HTTP connection handling
- Installer/distribution workflow
- Any handling of local files, config files, or credentials
